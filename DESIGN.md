# DESIGN.md

## 產品方向
Low Battery Pet 是一個以行動優先的情感陪伴應用。
核心體驗應該具備快速、平靜、可每日重複使用的特性。

## 目前階段
僅建立文件基線。
此階段不實作任何應用程式碼。

## MVP 範圍（規劃）
- 一個主要的行動優先應用介面
- 輕量的 check-in 互動
- 簡單的寵物心情/狀態回饋循環
- 清楚且簡潔的視覺語言

## 使用流程（規劃）
1. 開啟應用
2. 完成簡短 check-in
3. 取得寵物回應與更新後的心情狀態
4. 離開應用，狀態保留至下一次使用

## 架構（規劃）
- 純前端 React 應用
- Vite 建置工具
- TypeScript 提供型別安全
- styled-components 負責樣式管理
- Vitest + Testing Library 作為測試工具

## 本 Issue 非目標
- 不實作 UI
- 不加入後端或資料庫
- 不整合 AI 供應商服務
- 不加入登入或帳號系統
- 不實作 PWA 或離線能力

## 資料與狀態備註（規劃）
- 初始狀態模型保持簡單且明確。
- 早期 MVP 優先使用元件/模組內的本地狀態。
- 僅在流程複雜度提高時再引入共享/全域狀態。

## 品質準則
- 變更範圍必須受單一 Issue 約束。
- 優先清楚、可維護的結構，避免過度抽象。
- 使用最小且相關的檢查方式驗證變更。
